## 🔑 核心发现（一句话版）

| # | 发现 | 详细在哪 |
|---|------|---------|
| 1 | PHITS 7个有效列中只能恢复3个强列（1,7,8） | `06_PHITS真实数据分析/` |
| 2 | 动态α调度全面劣于固定α，被否决 | `02_方案A_动态alpha对比/` |
| 3 | 固定α EN 在极端条件下最稳健 | `02_方案A_动态alpha对比/` |
| 4 | HPP 可微化正确但小规模无优势 | `04_HPP可微ElasticNet/` |
| 5 | 贝叶斯EN + UQ 方向可行，待验证 | `05_贝叶斯ElasticNet/` |
| 6 | **κ≥30 时所有方法 NMSE > 0.88，条件数是决定性瓶颈** | `07_HPP条件数扫描/` |
| 7 | **Ridge 恢复率(86%)远高于 EN(70%)，但 NMSE 更差**——高恢复率 vs 高精度的策略选择 | `07_HPP条件数扫描/` |
| 8 | **超参优化仅改善恢复率 2-3%，瓶颈不在调参而在条件数** | `08_sparseho超参优化/` |
| 9 | **DogRot 预条件化：NMSE 从 6.79 降至 0.20（33×改善），但互相关无法改善** | `memory/2026-04-19.md` |
| 10 | **V14 框架锁定：BayesOpt (NMSE=0.024) 比 ElasticNetCV (0.133) 提升 82%** | `memory/2026-04-19.md` |
| 11 | **所有 EN 实验必须开启 fit_intercept=True（截距=本底辐射）** | `memory/2026-04-19.md` |

## 🔬 数据规格

- **PHITS 矩阵**：8探测器 × 12源位，7个有效列，cond=3.1e21
- **强列**：列1(22.6), 列7(9.3), 列8(38.5) → 可恢复
- **弱列**：列5(3.7e-5), 列6(1.1e-4), 列9(0.24), 列10(0.006) → 不可恢复
- **论文目标场景**：Cond~300 合成数据

## ⚠️ 被否决的方向

| 方向 | 否决原因 | 证据 |
|------|---------|------|
| 动态α调度 | MSE差1-3个数量级，支持恢复率<50% | 02_方案A/ 数据 |
| HPP替代EN | 小规模无优势，与EN持平 | 04_HPP/ 实验 |
| PHITS做核心验证 | cond=3.1e21远超论文场景 | 06_PHITS/ 诊断 |
| HPP EN 优势假设 | 10个κ下NMSE仅改善0.01-0.08，性价比不高 | 07_HPP条件数扫描/ 数据 |
| 超参优化突破瓶颈 | 网格搜索仅改善恢复率2-3%，瓶颈在条件数 | 08_sparseho超参优化/ 数据 |
| DogRot 解决根本问题 | 互相关系数=1.0未改善，只降条件数不降列相关性 | memory/2026-04-19.md |
