# 06 — PHITS 真实数据分析

## 要回答的问题

> PHITS 模拟得到的 8×12 系统矩阵，结构是什么样的？各种方法能恢复几个源？

## PHITS 矩阵结构

```
原始：8探测器 × 12源位
有效列：7个（列1,5,6,7,8,9,10）
零列：5个（列0,2,3,4,11）
有效子矩阵：7×7（去掉零行4）
条件数：3.1e21
```

## 列范数分布（决定了能不能恢复）

| 类型 | 原矩阵列号 | 列范数 | 能否恢复 |
|------|-----------|--------|---------|
| 🟢 强列 | 1 | 22.65 | ✅ 能 |
| 🔴 弱列 | 5 | 3.7e-5 | ❌ 不能 |
| 🔴 弱列 | 6 | 1.1e-4 | ❌ 不能 |
| 🟢 强列 | 7 | 9.26 | ✅ 能 |
| 🟢 强列 | 8 | 38.48 | ✅ 能 |
| 🟡 中等 | 9 | 0.239 | ❌ 勉强 |
| 🔴 弱列 | 10 | 0.006 | ❌ 不能 |

**结论：7个有效列中恢复3个强列（1,7,8）**

## 核心结论

1. PHITS 条件数 3.1e21，远超论文 Cond~300 场景
2. 所有方法 R² 均为负（极端欠条件）
3. 弱列物理上不可恢复（范数差 1e6 倍）
4. **PHITS 数据不适合做核心验证，应做"挑战案例"展示**
